Strategy Validation Documentation Plan

새 AI가 폴리마켓 온도 봇을 넘겨받아도 같은 철학으로 개발하게 만드는 문서 정리 기획서다.

Status: ready for explicit use Created: 2026-06-14 Asia/Seoul Paper-only strategy validation

대전제: 지금은 실거래 개발 단계가 아니다. 종이계좌 수익률이 실제 호가, 수수료, 정산 관측소, 데이터 지연을 반영한 믿을 만한 증거인지 확인하는 단계다.

이미 보호된 기반

온도 마켓만

비, 눈, 습도, 바람은 fail closed. trading-ready 도시만 실행한다.

실시간 호가

WebSocket을 메인으로 쓰고 REST는 bounded verification으로만 쓴다.

실제 체결가

진입은 ask-side VWAP, 청산은 bid-side VWAP를 쓴다.

가짜 청산 금지

bid depth가 없으면 HOLD_NO_LIQUIDITY지 CLOSE가 아니다.

온도 경계

exact는 표시값 그대로, range는 표시된 양끝값 그대로다.

장부 보호

paper_state.json과 paper_trades.csv는 서로 맞아야 한다.

조언 채택 판정

지금 채택

마켓 규칙 원문, station-local date, station grade, 정밀 비교, 최종 진입 검사, replayable evidence, report/dashboard 품질 지표.

나중 채택

도시별 calibration, Brier/LogLoss dashboard, region optimizer, large simulator, dynamic nowcast TTL.

거절

live key, midpoint PnL, 정산 근거 없는 도시 확장, retry bombing, exact half-step 복귀.

문서 역할

  • AGENTS.md: 헌법. 안정 규칙과 연구 태도만 둔다.
  • docs/production-decisions.md: 현행 법전. 지금 운영과 변경에 영향을 주는 규칙만 둔다.
  • docs/production-implementation-plan.md: 설계 지도. 코드 구조와 계약을 설명한다.
  • docs/strategy-validation-roadmap.md: 교과 과정. 전체 방향을 보여주되 활성 backlog가 아니다.
  • docs/active/current-task.md: 오늘의 숙제 카드. 진짜 미완료 작업이 있을 때만 active다.

다음 AI 작업 순서

  1. Market rule provenance and mismatch gate
  2. Station metadata quality grades
  3. Local event date windows
  4. Temperature comparison precision hardening
  5. Final pre-trade check and configurable spread guard
  6. Replayable decision and evidence snapshots
  7. Minimum performance report upgrade
  8. Dashboard exit liquidity view
  9. Confidence score and forecast freshness penalty
  10. Drawdown circuit breaker
  11. Portfolio payoff table and hidden overlap
  12. Paper-validation runbook and live readiness gates

정리 규칙: 완료된 작업은 current-task.md나 one-shot prompt에 남기지 않는다. 증거는 테스트, 커밋, 최종 응답, 또는 docs/solutions/에 둔다.

복사 프롬프트

AGENTS.md를 따라 진행해. 먼저 docs/active/current-task.md와 docs/production-decisions.md를 읽어. 이번에는 docs/strategy-validation-documentation-plan.md와 docs/plans/2026-06-14-001-strategy-validation-gap-closure-plan.md를 실행 기준으로 삼아 첫 미완료 작업부터 진행해. 첨부 조언을 무조건 베끼지 말고, 코드와 테스트를 확인해서 이미 보호된 것, 지금 채택할 것, 나중에 할 것, 버릴 것을 구분해. 현재 단계는 paper-only strategy validation이다. 실거래, 지갑, 개인키, 실주문, 상환, 카피트레이딩, LiveBroker는 금지다.
